# Contributing

Thank you for helping improve this project.

## Development Workflow

1. Open an issue that describes the workflow gap or bug.
2. Add or update a focused test.
3. Implement the smallest rule change that makes the test pass.
4. Run `npm test`.
5. Keep examples free of customer names, credentials, production records, and private documents.

## Pull Request Checklist

- Tests cover the workflow branch being changed.
- New terminology is understandable for multilingual ERP teams.
- No production data, database dumps, screenshots, secrets, or private files are included.
- The change is scoped to reusable OSS rules or documentation.

## Maintainer Review

Maintainers review for:

- Operational correctness
- Security and privacy hygiene
- Compatibility with customs brokerage and outsourced transport workflows
- Simplicity of adoption by small ERP teams
