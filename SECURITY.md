# Security Policy

Please do not open public issues for vulnerabilities that include private data, credentials, customer records, or exploitable production details.

Report security concerns by opening a private advisory on GitHub when available, or by contacting the maintainer through the public GitHub profile.

## Supported Scope

This repository contains workflow rules and documentation. It should not contain:

- API keys
- Passwords
- Customer records
- Production database files
- Uploaded commercial documents
- Private screenshots

If any sensitive material is found, open a security report immediately so it can be removed from history.
